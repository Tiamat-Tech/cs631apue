Using git(1)

Summary

Learning to use a code revision system and collaborating with others is an important aspect of programming. Git is a popular free and open source distributed version control system and well worth knowing at least the fundamental of.

Note: git(1) can be used without and independent of GitHub! In this class, we will use git(1) on cs631apue.netmeister.org for your final project.

Accessing the repository

Your instructor will set up the repository for your group. You will then access the repository via ssh(1) under the path cs631apue.netmeister.org:/usr/local/apue/$team (where $team is the name of your team).

For example, if your username on cs631apue.netmeister.org was 'jdoe' and you were a member of the team 'hyper', you would check out the repository using the following commands: 

$ git clone jdoe@cs631apue.netmeister.org:/usr/local/apue/hyper

Working with git

You should make it a habit to begin your working session by updating your copy of the repository and ending it by committing your changes. If you are ready to push your changes to make them available to your team mates, you would then push them to the repository: 

$ cd hyper
$ git pull
$ $EDITOR http.c
[edit, debug]
$ git commit
$ git push

Please always write a meaningful commit message. See these links for why/how:

  • Commit messages - Good practices
  • 5 Useful Tips For A Better Commit Message
  • A Note About Git Commit Messages
