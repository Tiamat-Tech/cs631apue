Advanced Programming in the UNIX Environment

Pipes Practice

In order to better understand Unix systems, it's best to actually use Unix systems. One way to do that is to practice using pipelines to accomplish certain tasks, to get used to invoking the common Unix tools, like sed(1), grep(1), awk(1), sort(1), uniq(1), head(1), tail(1), ...

So let's play a game of Pipes Practice, inspired by the UNIX Pipe Card Game and an exercise I use in CS615 - System Administration:

Let's suppose we have copy-pasta'd the IMDB Top 250 Movies page into a plain text file. Unfortunately, the formatting got all messed up, so now the file looks like this:

Each week, we'll pose a "challenge" to construct a pipeline that uses this data file as input and that should provide the requested output. There are two challenge modes:

  1. Find the shortest command or pipeline to produce the desired output.
  2. Find the weirdest, most creative, absurd, or simply most convoluted command or pipeline to produce the desired output.

Note: you may not install additional software; all challenges must be completed using only commands shipped with NetBSD in the default installation.

For the second challenge, no command may be invoked more than once (e.g., "cat | cat | cat" or "base64 | base64 -d" is not allowed).

I will post the weekly challenge in Slack, and upon resolution update this page with the winning submissions.

Unless otherwise noted, the data file to use as input for all challenges is this file.
